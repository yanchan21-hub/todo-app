# ToDoリスト

シンプルで使いやすいToDoリストWebアプリです。

## 使用技術

| カテゴリ | 技術 |
|---|---|
| フレームワーク | Next.js 15 (App Router) |
| 言語 | TypeScript |
| スタイリング | Tailwind CSS |
| 状態管理 | React Hooks (useState / useEffect) |
| データ永続化 | localStorage |

## 機能

- **タスクの追加** — 入力欄にタスクを入力して「追加」ボタンで登録
- **完了/未完了の切り替え** — チェックボタンを押すと完了状態をトグル
- **タスクの削除** — 「削除」ボタンでタスクを一件削除
- **データ永続化** — ブラウザを閉じても localStorage でデータを保持

## 工夫した点

### UX
- 入力が空のまま追加できないよう、「追加」ボタンを `disabled` 制御
- タスクがゼロのとき「まだタスクがありません」の案内文を表示し、操作に迷わないように配慮
- 残りタスク件数をリスト上部に表示し、進捗を把握しやすくした
- 全完了時は「全てのタスクが完了しました 🎉」と達成感のあるメッセージを表示

### アクセシビリティ・スマホ対応
- 文字サイズは最小16pxを維持し、スマホでも読みやすく
- ボタンはタップしやすい十分な大きさ（padding: py-3 以上）を確保
- `aria-label` をボタンに付与し、スクリーンリーダーに対応

### 完了状態の視覚的区別
- **チェックマーク**（緑の丸ボタン内の ✓）
- **取り消し線**（`line-through`）
- **文字色のグレーアウト**（`text-gray-400`）

の3つを組み合わせて完了タスクを一目で識別できる

### SSR対応
- `storage.ts` で `typeof window === 'undefined'` チェックを入れ、Next.js のサーバーサイド実行時にクラッシュしないよう対応
- `page.tsx` の `mounted` フラグで、ハイドレーション前の描画と localStorage 読み込み後の描画を分離

## 起動方法

```bash
npm install
npm run dev
```

ブラウザで http://localhost:3000 を開いてください。
